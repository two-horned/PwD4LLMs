** (Brzozowski) Derivative
*** 
:PROPERTIES:
:BEAMER_env: definition
:BEAMER_act: <1->
:END:
The derivative of a language w.r.t. letter $u$
1) filter for words beginning with $u$
2) "cut away" first letter
\[ D_u^{}S = \{ v \in \Sigma^* \mid uv \in S \} \]
@@latex: { \raggedleft \tiny \cite{brz:derivative}, \cite{might:pwd}, \cite{moss:peg-derivative}}@@

*** Example
:PROPERTIES:
:BEAMER_env: exampleblock
:BEAMER_act: <2->
:END:
Derivative on month names w.r.t. letter $a$.
\[ D_\mathbf{a}^{}\{ august, september, april \} = \{ ugust, pril \} \]

** Parsing with Derivatives
Parsing reduced down to (repeated) application of the derivative (given respective input symbol).

#+BEGIN_src scala

  def feed[R](p: Parser[R], in: Elem): Parser[R]
  def feedAll[R](p: Parser[R], in: Iterable[Elem]): Parser[R] =
    in.foldLeft(p)(feed)

  // fcd: library/DerivativeParsers.scala
  def parse[R](p: Parser[R], in: Iterable[Elem]): Results[R] =
    List.from(feedAll(p, in).results)

** Parser
The parser allows token input verification.
#+BEGIN_src scala
trait Parser[T, R] {
  def feed(t: T): Parser[T, R]
  def result: R
  def state: ParserState
}
#+END_src
- @@b:<2->@@ consumes tokens of type \textbf{T} to create new parser
- @@b:<3->@@ returns (partial) result of type \textbf{R}
- @@b:<4->@@ has a state (\textcolor{Orange}{pending}, \textcolor{ForestGreen}{accepting}, \textcolor{BrickRed}{failed})

** Parser States
The state of the parser.
#+BEGIN_src scala
  enum ParserState {
    case Accepting
    case Pending
    case Failed
  }
#+END_src
- @@b:<2->@@ \textcolor{Orange}{pending}: parser does \textbf{not} accept, children may be in any state
- @@b:<3->@@ \textcolor{ForestGreen}{accepting}: parser does accept \textit{currently}, children may be in any state
- @@b:<4->@@ \textcolor{BrickRed}{failed}: parser does \textbf{not} accept, children must also be failed

** Useful Parsers
Wrap a derivative parser built from \textbf{fcd} parser combinators.
#+BEGIN_src scala
trait DerivativeParserTools[P <: fcd.DerivativeParsers](val parsers: P) {
  class WrappedParser[R](
      inner: parsers.Parser[R]
  ) extends Parser[parsers.Elem, Iterable[R]] {
    def feed(t: parsers.Elem) = WrappedParser(inner.consume(t))
    def result = inner.results
    def state =
      if inner.accepts then Accepting
      else if inner.failed then Failed
      else Pending
  }

  type DParser[R] = parsers.Parser[R]
}
#+END_src
- @@b:<2->@@ pass parser combinator object \texttt{parsers} to create tools
- @@b:<3->@@ wrap parser with \texttt{WrappedParser} use as \textbf{Parser} type


** Token Generator
The token generator defines a (dynamic) search tree of token inputs.
#+BEGIN_src scala
trait TokenGenerator[T] {

#+END_src

** Generator Actions
The actions defined on the current token input.
#+BEGIN_src scala
enum GeneratorAction[Token] {
  case Concatenate(tokens: Iterator[Token])
  case Append(token: Token)
  case DropLast(number: Int)
  case DeleteLast()
  case ReplaceLast(token: Token)
  case Rebuild(tokens: Iterator[Token])
  case Finish()
  case Reset()
}
#+END_src
Mostly self-explanatory. Special editing commands:
- @@b:<2->@@ \texttt{rebuild}: rebuild the whole input with new tokens.
- @@b:<3->@@ \texttt{reset}: reset the current input stream.
- @@b:<4->@@ \texttt{finish}: stop or abort editing.

** EvalResult
The result of an evaluation.
The (partial) result field is a function to avoid expensive computation, if not needed.
#+BEGIN_src scala
enum EvalResult[R] {
  case Success(result: () => R)
  case Failure(partial_result: () => R)
  case CriticalFailure()
}
#+END_src

- @@b:<2->@@ \textcolor{ForestGreen}{Success},
  when everything goes well
- @@b:<3->@@ \textcolor{BrickRed}{Failure},
  when token generator finds no accepting state
- @@b:<4->@@ \textcolor{Maroon}{CriticalFailure},
  when token generator does something silly (e.g., \texttt{delete-last} on empty token input)

** Useful Evaluators
Choosing right evaluator depends on requirements, i.e. the token generator used.
# | Evaluator               | Internal Datastructures            | Example Usecase                       |
# |-------------------------+------------------------------------+---------------------------------------|
# | StackEvaluator          | Stack of parsers                   | DFS search                            |
# | ScrapAllEvaluator       | Stack of tokens (for backup)       | No backtracking logic needed          |
# | RememberActionEvaluator | Stack of parsers (for each action) | Backtracking (big) reversible actions |
# |                         | Stack of tokens (for backup)       | (i.e. concat many tokens)             |

- @@b:<2->@@ \textbf{StackEvaluator}.
  Internals: stack of parsers created for each token.
  Good for backtracking in DFS.
- @@b:<3->@@ \textbf{ScrapAllEvaluator}.
  Internals: current token input.
  Backtracking requires parsing everything anew.
  Good for search not requiring backtracking.
- @@b:<4->@@ \textbf{RememberActionEvaluator}.
  Internals: parsers created by \texttt{GeneratorAction}s.
  Good for backtracking (big) reversible actions
  (i.e. concat many tokens).

** Token Generation (3)
In more detail: Dynamic weight adjustion in specified direction
(up \rightarrow more token vs. down \rightarrow less tokens needed).
#+BEGIN_src scala
    private val steps = 4  // how "aggressive" adjustment is
    private val shift = 11 // correction, cuz expr inherently tend to grow
    final def up(bias: Int): Int = {
      val b = steps * budget - shift
      (b * max(b, 0) + 1) * bias
    }
    final def down(bias: Int): Int = {
      val b = steps * budget - shift
      (b * min(b, 0) + 1) * bias
    }
#+END_src
- @@b:<2->@@ \textbf{Budget}: desired token length. 
- @@b:<3->@@ \textbf{Steps}: weight of adjustment. 
- @@b:<4->@@ \textbf{Bias}: for relative likelihood of expression.
- @@b:<5->@@ Experimentally verified.
